# SEO Priorities

Always optimize in this order:

1. Content quality
2. Semantic HTML
3. Information architecture
4. Metadata
5. Structured data
6. Performance

Never optimize purely for search engines at the expense of the user experience.

---

# Content

Every page should target a single primary topic.

Ensure:

- one clear search intent
- descriptive headings
- meaningful content
- concise copy
- natural keyword usage

Avoid:

- duplicate content
- keyword stuffing
- filler text
- generic headings

Write for humans first.

---

# Semantic HTML

Use semantic HTML wherever possible.

Prefer:

- header
- nav
- main
- section
- article
- aside
- footer

Use semantic elements before relying on ARIA.

A clear document structure improves both accessibility and SEO.

---

# Heading Hierarchy

Headings define the structure of the page.

Always:

- use exactly one `<h1>` per page
- maintain a logical heading hierarchy
- organize content into meaningful sections

Avoid skipping heading levels without reason.

---

# Metadata

Every page should provide unique metadata.

Ensure:

- descriptive title
- unique meta description
- canonical URL when needed
- Open Graph metadata
- Twitter Card metadata

Titles should accurately describe the page.

Descriptions should encourage users to click without becoming misleading.

---

# URLs

URLs should be:

- descriptive
- readable
- stable
- lowercase
- human-friendly

Avoid:

- unnecessary parameters
- random IDs
- long URLs
- meaningless slugs

---

# Structured Data

Use structured data whenever it provides additional context.

Prefer appropriate Schema.org types.

Examples include:

- Organization
- Website
- Article
- Product
- BreadcrumbList
- FAQ
- LocalBusiness

Only add structured data that accurately reflects the page.

Never generate misleading structured data.

---

# Images

Images contribute to SEO.

Always:

- provide meaningful alt text
- optimize file size
- use descriptive filenames
- reserve layout space

Decorative images should use empty alt attributes.

---

# Internal Linking

Build a logical internal linking structure.

Ensure:

- related pages link to one another
- important pages remain easy to reach
- anchor text describes the destination

Avoid generic links like:

- Click here
- Learn more

when a more descriptive alternative is available.

---

# Crawlability

Ensure search engines can properly crawl the website.

Verify:

- robots.txt
- sitemap.xml
- canonical URLs
- correct indexing rules

Prevent duplicate content whenever possible.

---

# Performance

Performance directly affects SEO.

Prioritize:

- fast loading
- stable layouts
- responsive interactions
- optimized assets

Maintain excellent Core Web Vitals.

---

# Accessibility

Good accessibility improves SEO.

Ensure:

- semantic HTML
- descriptive labels
- meaningful link text
- proper heading structure
- keyboard accessibility
- readable content

Accessibility and SEO should reinforce each other.

---

# Multi-Device Experience

Every page should work consistently across devices.

Verify:

- responsive layouts
- readable typography
- touch-friendly interactions
- mobile navigation

Design mobile-first.

---

# Technical Quality

Avoid technical issues that reduce search visibility.

Prevent:

- broken links
- duplicate metadata
- orphan pages
- redirect chains
- incorrect canonical tags
- missing alt attributes
- invalid structured data

---

# SEO Checklist

Before completing a page, verify:

- Unique title
- Unique meta description
- One `<h1>`
- Logical heading hierarchy
- Semantic HTML
- Correct structured data
- Optimized images
- Internal links
- Responsive layout
- Fast loading
- No crawlability issues

---

# Anti-Patterns

Avoid:

- Keyword stuffing.
- Duplicate content.
- Duplicate metadata.
- Multiple `<h1>` elements.
- Generic page titles.
- Generic meta descriptions.
- Broken links.
- Missing image alt text.
- Hidden content intended only for search engines.
- Automatically generated low-quality content.
- Misleading structured data.
- Pages without a clear purpose.
